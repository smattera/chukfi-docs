## Why Chukfi?

Chukfi is a headless CMS built from the ground up for **speed, simplicity, and AI-native workflows**. Three non-negotiable pillars:

<div class="feature-grid">

<div class="feature-card">
<div class="feature-icon terminal">></div>
<h3>CLI-First Distribution</h3>
<p>The Rust binary ships as platform-specific npm packages. <code>npx chukfi</code> — no Rust toolchain, no build steps, no Docker Compose files to write.</p>
</div>

<div class="feature-card">
<div class="feature-icon database">DB</div>
<h3>Local-to-Prod Parity</h3>
<p>Docker PostgreSQL locally, AWS RDS in production. Same engine, same migrations, same SQL. No SQLite surprises at deploy time.</p>
</div>

<div class="feature-card">
<div class="feature-icon cloud">☁</div>
<h3>One-Command Deploy</h3>
<p><code>npx chukfi deploy</code> provisions the full AWS stack via CDK. One domain, one bill, no hybrid hosting. Free Tier eligible.</p>
</div>

</div>

## AI-Native by Design

Every CLI command outputs structured JSON. This means **AI agents can operate Chukfi as easily as humans** — no screen scraping, no brittle parsing.

<div class="feature-grid">

<div class="feature-card accent">
<div class="feature-icon sparkle">✦</div>
<h3>Teams + Bedrock Bot</h3>
<p>Connect Claude 3.5 Haiku via Microsoft Teams. Staff can create, list, and update content from chat — all forced to draft for human review.</p>
<p><a href="/teams-bot/overview/">Learn more →</a></p>
</div>

<div class="feature-card accent">
<div class="feature-icon sparkle">✦</div>
<h3>JSON-Output CLI</h3>
<p>Every command accepts <code>--json</code> for machine-parseable responses. AI agents get structured data, not terminal tables.</p>
</div>

<div class="feature-card accent">
<div class="feature-icon sparkle">✦</div>
<h3>Draft-Only Safety</h3>
<p>AI agents can create and edit content, but never publish. Every change requires human review — enforced in code, not prompts.</p>
</div>

</div>

## Get Started in 30 Seconds

```bash
npm install @chukfi/cli
npx chukfi dev
```

That's it. Chukfi detects Docker, spawns PostgreSQL, runs migrations, and starts the API on port 8080. Your CMS is live.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| **Backend** | Rust (native binary via Axum) |
| **Frontend** | Astro 7 + React islands |
| **Database** | PostgreSQL (Docker local / AWS RDS) |
| **Auth** | Magic Link + Microsoft Entra ID |
| **AI** | Amazon Bedrock (Claude 3.5 Haiku) |
| **Media** | S3 / Local filesystem |
| **Deploy** | AWS CDK / Cloudflare Pages |
